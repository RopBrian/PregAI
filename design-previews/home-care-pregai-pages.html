The Home Care direction, expanded into real product pages.

This keeps your current component map and functionality, but changes the visual language: fewer repeated cards, more full-width bands and divided sections, real human photography, soft clinical warmth, and button/message labels that sound like a person.

Design thesis: PregAI should feel like a calm care companion at home, not a cold diagnostic machine. The AI is present, but the mother’s understanding, privacy, and next step are the center of the interface.

Landing Page

Uses one strong human image, direct reassurance, and feature bands instead of a stack of cards. Header maps to the existing `Header` and `LandingPage` structure.

Implementation Translation

To implement this in the actual React frontend, the safest path is mostly CSS and markup structure: keep API logic and state behavior intact, retune component labels, replace repeated glass cards with section dividers and bands, add licensed human imagery to `src/assets`, and create shared design tokens in `global.css` for the Home Care system.
